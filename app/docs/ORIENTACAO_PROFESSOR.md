# Orientação de aplicação em aula

## Objetivo

Concluir uma integração visual mínima para que os acadêmicos consigam demonstrar o fluxo funcional do AtendeLab dentro do tempo disponível.

## Recorte sugerido

Priorizar em sala:

1. Login e dashboard.
2. Listagem e cadastro de pessoas.
3. Listagem e cadastro de tipos.
4. Registro de atendimento.
5. Alteração de status com observação final.

Deixar como evolução opcional:

- edição visual completa de atendimento;
- filtros avançados;
- relatórios;
- paginação;
- dashboard analítico;
- layout definitivo;
- CRUD visual de usuários.

## Ponto pedagógico

As telas não substituem o backend. Elas permitem visualizar o fluxo já estudado:

```text
View → rota → controller → validação → PDO → MySQL → resposta → View
```

Antes de copiar, cada aluno deve comparar nomes de ações e campos com o próprio projeto.
